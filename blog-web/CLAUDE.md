# Claude Code 开发流程规范

## 核心原则
- 产品经理（我）负责需求把控
- Claude Code 负责执行开发
- 每个步骤都需要我审核后才能继续

---

## 标准开发流程

### 第一步：Plan 计划
1. Claude Code 分析需求，制定实现方案
2. 输出 Plan，包含：
   - 功能拆解
   - 技术方案
   - 文件改动清单
   - 风险点
3. **等待产品经理审核** → 我确认后 Claude Code 才能执行

### 第二步：执行开发
1. 按照审核通过的 Plan 执行开发
2. 每完成一个模块，输出进度说明
3. 遇到问题及时反馈，不要闷头做

### 第三步：代码审核
1. 开发完成后，进行自我代码审核
2. 检查：
   - 代码规范
   - 潜在 Bug
   - 安全性
   - 性能影响
3. 输出审核报告

### 第四步：执行测试
1. 运行 `npm run build` 验证构建
2. 运行 `npm run type-check` 验证类型
3. 如有测试用例，执行测试
4. 输出测试结果

### 第五步：问题修复
- 如有问题 → 记录问题 → 修复 → 返回第一步重新审核
