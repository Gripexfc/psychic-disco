## Chrome-DevTools MVP 走查报告

### 走查范围

- 首页浏览、文章列表、搜索筛选、文章详情
- 登录（成功/失败）、权限拦截、发布文章、退出登录
- 控制台信息与网络请求结果核验

### 关键结论

- 核心流程可走通：登录、发布、列表回显、详情访问、退出均可完成。
- 发现若干功能与体验缺陷，主要集中在可用性细节、国际化兼容和可访问性。

### 问题清单（按优先级）

#### 严重（S2）

1. **中文标题生成 slug 质量差，存在可读性和潜在冲突风险**
   - 现象：发布标题为中文时，详情路由变为类似 `#/posts/devtools-`。
   - 影响：URL 不可读，且中文标题在 `toSlug()` 中被大量剔除，后续可能出现冲突或 SEO 不友好。
   - 关联：`backend/src/index.js`（`toSlug` 逻辑与发布接口）
   - 建议：改为支持 Unicode slug（拼音/转码）或使用稳定 ID 作为主路由键，slug 仅做展示。

#### 一般（S3）

2. **未登录访问发布页时，登录页与权限弹窗同时出现，信息重复**
   - 现象：点击“发布/发布新文章”后跳到 `#/login`，同时出现“权限受限”弹窗。
   - 影响：用户感知重复提醒，流程显得冗余。
   - 关联：`src/App.vue`（路由守卫 + 权限弹窗触发）
   - 建议：二选一策略：仅跳转登录页并提示，或停留当前页弹窗引导，不同时出现。

3. **登录表单存在可访问性问题（控制台 issue）**
   - 现象：DevTools 提示表单字段缺少 `id/name`，密码框缺少 `autocomplete`。
   - 影响：浏览器自动填充、无障碍体验和表单语义受影响。
   - 关联：`src/App.vue` 登录表单
   - 建议：补齐 `id/name`、`autocomplete="username|current-password"`。

#### 优化项（S4）

4. **发布页工具栏操作项偏多，移动端首屏阅读压力较大**
   - 现象：编辑器工具栏按钮数量较多，窄屏下信息密度偏高。
   - 影响：首屏视觉拥挤，新手用户学习成本较高。
   - 关联：`src/components/RichTextEditor.vue`
   - 建议：增加“更多”分组或两级工具栏；常用功能前置，低频功能折叠。

5. **权限相关反馈文案可进一步统一**
   - 现象：Toast、弹窗、页面文本在“需要登录”场景有多套描述。
   - 影响：一致性略弱。
   - 关联：`src/App.vue`、`src/components/AppModal.vue`
   - 建议：抽离统一权限文案常量，保证语气与动作一致。

### 网络与控制台核验摘要

- 登录失败请求：`POST /api/auth/login` 返回 `401`（符合预期）
- 登录成功请求：`POST /api/auth/login` 返回 `200`
- 发布请求：`POST /api/posts` 返回 `201`
- 退出请求：`POST /api/auth/logout` 返回 `200`
- 控制台 issue：
  - Input 缺少 `autocomplete`
  - 表单字段缺少 `id/name`（2 处）

### 下一步建议

1. 先修复 S2（slug 生成策略），避免后续内容增长后路由质量恶化。  
2. 处理 S3 登录流程冗余与表单可访问性，提升基础体验一致性。  
3. 再执行一轮同路径复测，确认无回归后进入视觉打磨阶段。
