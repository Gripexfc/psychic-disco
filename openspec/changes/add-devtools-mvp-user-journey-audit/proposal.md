## Why

当前功能迭代较快，但缺少一套以真实用户视角进行端到端走查的机制，容易遗漏流程断点、交互阻力和视觉不友好问题。需要通过 chrome-devtools MVP 跑通全链路，系统化发现并修复体验缺陷。

## What Changes

- 使用 chrome-devtools MVP 对网站关键路径进行真实用户操作模拟（访问、浏览、登录、发布、退出等）。
- 覆盖所有核心功能流程，记录功能缺陷、交互不友好点、视觉不一致或可读性问题。
- 建立统一问题分级标准（阻断、严重、一般、优化项）与修复优先级。
- 形成可执行改进清单，并将问题闭环到具体页面与组件。
- 输出复测机制，确保修复后问题不回归。

## Impact

- 主要影响 QA 流程、体验改进节奏与前端交互/视觉细节优化策略。
- 短期增加走查与修复成本，长期显著降低线上体验问题和重复返工。
- 为后续版本建立可持续的“用户旅程质量基线”。
