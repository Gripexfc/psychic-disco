## Why

当前博客前端已具备页面结构，但缺少真实后端接口，无法进行完整联调与数据流验证。需要新增一个后端项目，提供稳定、易理解、可直接被前端调用的 API，降低联调成本并提升迭代效率。

## What Changes

- 新增一个独立后端服务，面向当前博客提供内容接口（文章列表、文章详情、标签、站点信息等）。
- 设计前端友好的 API 规范：统一响应结构、统一错误码、分页参数一致、字段命名清晰。
- 提供本地开发联调能力：跨域配置、环境变量示例、启动说明、请求示例。
- 提供最小可用鉴权扩展位（可选），确保后续新增管理端能力时可平滑扩展。
- 增加接口文档与示例数据，保障前端同学可以不依赖后端口头说明独立接入。

## Impact

- 新增后端工程与接口文档维护成本，但可显著降低前后端沟通摩擦。
- 前端将从静态数据切换到真实 API 调用，联调与异常处理路径更完整。
- 为后续扩展（搜索、评论、后台发布）奠定统一接口基础。
