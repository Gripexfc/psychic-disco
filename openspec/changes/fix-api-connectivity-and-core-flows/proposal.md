## Why

当前系统存在“接口未走通”风险，导致首页数据展示和文章上传流程不稳定，影响基础可用性与联调效率。需要优先修复接口连接链路，确保关键路径稳定可用。

## What Changes

- 排查并修复前后端接口未走通问题（包含地址配置、代理转发、跨域、环境变量和请求路径一致性）。
- 增加首页数据加载兜底策略，确保在接口异常时也有明确反馈与可恢复路径。
- 修复并加固文章上传流程，确保富文本发布、参数校验、提交状态与错误提示完整可用。
- 统一关键接口的成功/失败响应处理逻辑，避免页面出现静默失败。
- 补充联调验证清单，覆盖首页加载、文章列表、文章详情和文章上传全流程。

## Impact

- 主要影响前端请求层、页面状态管理、后端接口行为与联调配置。
- 不改变既定业务范围，但显著提升系统稳定性与开发调试效率。
- 为后续功能迭代提供可复用的“接口可用性基线”。
